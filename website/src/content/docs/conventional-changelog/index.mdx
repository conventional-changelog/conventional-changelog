`conventional-changelog` reads your git history and turns it into a `CHANGELOG.md`, grouping commits into sections like **Features** and **Bug Fixes**. You write meaningful commit messages following a convention, and the tool assembles the release notes for you.

To do this, the tool needs to understand the *shape* of your commit messages. That structure comes from a **commit convention** — most commonly [Conventional Commits](https://www.conventionalcommits.org/) — which is applied through a [preset](#available-presets).

## Quick start

<Steps>

1. **Install `conventional-changelog`:**

   <Tabs syncKey="pkg">
     <TabItem label="pnpm">
       ```sh frame="none"
       pnpm add -D conventional-changelog
       ```
     </TabItem>
     <TabItem label="npm">
       ```sh frame="none"
       npm i -D conventional-changelog
       ```
     </TabItem>
     <TabItem label="yarn">
       ```sh frame="none"
       yarn add -D conventional-changelog
       ```
     </TabItem>
   </Tabs>

2. **Choose a preset.** A preset teaches the tool how to parse and group your commits. Since we're following [Conventional Commits](https://www.conventionalcommits.org/), install the matching preset:

   <Tabs syncKey="pkg">
     <TabItem label="pnpm">
       ```sh frame="none"
       pnpm add -D conventional-changelog-conventionalcommits
       ```
     </TabItem>
     <TabItem label="npm">
       ```sh frame="none"
       npm i -D conventional-changelog-conventionalcommits
       ```
     </TabItem>
     <TabItem label="yarn">
       ```sh frame="none"
       yarn add -D conventional-changelog-conventionalcommits
       ```
     </TabItem>
   </Tabs>

   Following a different convention? See all [available presets](#available-presets).

3. **Make commits** following the convention as you work:

   ```
   feat(api): add async write() generator
   fix(cli): resolve config path relative to cwd
   perf(parser): cache compiled header regex
   ```

4. **Run** the generator with your preset. The `conventionalcommits` alias resolves to the `conventional-changelog-conventionalcommits` package you just installed:

   <Tabs syncKey="pkg">
     <TabItem label="pnpm">
       ```sh frame="none"
       pnpm conventional-changelog -p conventionalcommits
       ```
     </TabItem>
     <TabItem label="npm">
       ```sh frame="none"
       npx conventional-changelog -p conventionalcommits
       ```
     </TabItem>
     <TabItem label="yarn">
       ```sh frame="none"
       yarn conventional-changelog -p conventionalcommits
       ```
     </TabItem>
   </Tabs>

   Add `--stdout` to print the result to the terminal instead of writing the file.

5. **Get your changelog.** A new section is prepended to `CHANGELOG.md` (existing entries are kept), using the `version` from your `package.json`:

   ```markdown title="CHANGELOG.md"
   ## [1.1.0](https://github.com/acme/app/compare/v1.0.0...v1.1.0) (2026-07-01)

   ### Features

   * **api:** add async write() generator ([0f7e2c1](https://github.com/acme/app/commit/0f7e2c1a9d3e4b5c6f7089abcdef0123456789ab))

   ### Bug Fixes

   * **cli:** resolve config path relative to cwd ([a3b9d84](https://github.com/acme/app/commit/a3b9d8472e1f0c9b8a7d6e5f4c3b2a1908f7e6d5))

   ### Performance Improvements

   * **parser:** cache compiled header regex ([c1d2e3f](https://github.com/acme/app/commit/c1d2e3f4a5b6c7d8e9f0a1b2c3d4e5f6a7b8c9d0))
   ```

</Steps>

By default the version comes from the closest `package.json`, only commits made **since the last semver tag** are included, and the result is written to `CHANGELOG.md`. When `package.json` has a `repository` field, versions and commit hashes link to your git host (as above); without it, they are plain text.

## Available presets

A preset maps a commit convention to the parsing rules and changelog layout the tool uses. Install the one that matches your commits and pass its name to `--preset` (or `-p`):

| Preset | Package | Convention |
| --- | --- | --- |
| [`conventionalcommits`](/presets/conventional-commits/) | `conventional-changelog-conventionalcommits` | [Conventional Commits](https://www.conventionalcommits.org/) |
| [`angular`](/presets/angular/) | `conventional-changelog-angular` | [Angular commit convention](https://github.com/angular/angular/blob/main/contributing-docs/commit-message-guidelines.md) |

## Common tasks

### Regenerate the whole changelog

The first time you adopt the tool, generate notes for **every** release at once. `--release-count 0` (or `-r 0`) rebuilds the full history and **overwrites** the output file:

<Tabs syncKey="pkg">
  <TabItem label="pnpm">
    ```sh frame="none"
    pnpm conventional-changelog -p conventionalcommits -r 0
    ```
  </TabItem>
  <TabItem label="npm">
    ```sh frame="none"
    npx conventional-changelog -p conventionalcommits -r 0
    ```
  </TabItem>
  <TabItem label="yarn">
    ```sh frame="none"
    yarn conventional-changelog -p conventionalcommits -r 0
    ```
  </TabItem>
</Tabs>

### Generate for a specific range

Limit generation to a commit range with `--from` and `--to` (each accepts a tag or SHA):

<Tabs syncKey="pkg">
  <TabItem label="pnpm">
    ```sh frame="none"
    pnpm conventional-changelog -p conventionalcommits --from v1.0.0 --to HEAD
    ```
  </TabItem>
  <TabItem label="npm">
    ```sh frame="none"
    npx conventional-changelog -p conventionalcommits --from v1.0.0 --to HEAD
    ```
  </TabItem>
  <TabItem label="yarn">
    ```sh frame="none"
    yarn conventional-changelog -p conventionalcommits --from v1.0.0 --to HEAD
    ```
  </TabItem>
</Tabs>

<Aside title="See all options">
  Every flag is listed by running `conventional-changelog --help`, and documented on the [CLI](/conventional-changelog/cli/) page.
</Aside>

## Programmatic usage

Prefer to generate changelogs in code? The same result is available through the [JS API](/conventional-changelog/api/):

```js
import { ConventionalChangelog } from 'conventional-changelog'

const generator = new ConventionalChangelog()
  .readPackage()
  .loadPreset('conventionalcommits')

for await (const chunk of generator.write()) {
  process.stdout.write(chunk)
}
```
